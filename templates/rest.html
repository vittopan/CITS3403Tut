REST

(Representational State Transfer) are Architectures for the web that describe interactions with web based resources.

API

Application Programming interface are used so two websites can communicate over the internet.

1. Client Server Model

Seperates the Roles within the server, The client and the server communicate throught the HTTP Requests and TCP/IP

2. Layered System

States that there doesn't need to be a direct link between the client and the server rather it can use intermediate nodes through proxy chaining.

3. Cache

States that the user can use local browser cache to store previous HTTP requests, so the user does not need to access the server every time Good for efficiency

4. Code On Demand

States that the server can provide executable code in response to a request.

5. Stateless

Statelessness is a key property in HTTP, memory of prior transactions should not be saved and a request must contain sufficient context to receive a response.

6. Uniform Interface

Uniform Interface is used so that the client does not have to create a new UI everytime they want to access the server.
