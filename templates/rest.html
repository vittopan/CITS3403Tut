REST

(Representational State Transfer) are Architectures for the web that describe interactions with web based resources.

API

Application Programming interface are used so two websites can communicate over the internet.

Six Principles of REST API

1. Client Server Model

Seperates the Roles within the server, The client and the server communicate throught the HTTP Requests and TCP/IP

2. Layered System

States that there doesn't need to be a direct link between the client and the server rather it can use intermediate nodes through proxy chaining.

3. Cache

States that the user can use local browser cache to store previous HTTP requests, so the user does not need to access the server every time Good for efficiency

4. Code On Demand

States that the server can provide executable code in response to a request.

5. Stateless

Statelessness is a key property in HTTP, memory of prior transactions should not be saved and a request must contain sufficient context to receive a response.

6. Uniform Interface

Uniform Interface is used so that the client does not have to create a new UI everytime they want to access the server.

RESTful API works and CRUD actions

The RESTful API works together with HTTP verbs to map URLS to perform the CRUD actions.

Create(POST)

  • Collection: Adding a new 'book' to the collection
  • HTTP Method: `POST`
  • Request Body:
    • {
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1925-04-10"
    }
  • Response: The API creates a new book entry and returns the details of the created book, often including a unique identifier.
    • {
    "id": 1,
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1925-04-10"
    }

READ (GET)

  • Collection: Retrieving the list of all books.
  • HTTP Method: `GET`
  • Response:
    • [ {
    "id": 1,
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1925-04-10"
    }, { "id": 2,
    "title": "To Kill a Mockingbird",
    "author": "Harper Lee",
    "published_date": "1960-07-11"
    } ]
  • Individual Element: Retrieving details of a specific book by its ID
  • HTTP Method`GET`
  • Response:
    • {
    "id": 1,
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1925-04-10"
    }

UPDATE(PUT/PATCH)

  • Individual Element: Updating the details of a specific book.
  • HTTP Method: `PUT` (or `PATCH` for partial updates)
  • Request Body:
    • {
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1926-04-10"
    }
  • Response: The API updates the specified book and returns the updated details.
    • {
    "id": 1,
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "published_date": "1926-04-10"
    }

DELETE(DELETE)

  • Individual Eleement: Deleting a specific book by its ID.
  • HTTP Method: `DELETE`
  • Response: The API deletes the specified book, often returning a status message.
    • { "message": "Book deleted successfully" }
