Server Side and Client Side

Server Side (RSRR)

SSR involves generating the HTML content on the server and sending it to the client when it is a fully formed webpage.

  1. Request Handling: The client makes a request to the server.
  2. Server Processing: The server processes the request, executes any necessary logic, and generates the complete HTML.
  3. Response: The server sends the complete HTML and displays the web page.
  4. Rendering: The client's browser renders the received HTML and displays the web page.

Client Side (RACR)

CSR involves generating the HTML content in the client's browser using JavaScript. This approach typically uses a REST API, AJAX, and jQuery for making HTTP requests and dynamically updating the web page.

  1. Request: The client requests a basic HTML shell from the server.
  2. Execution: The basic HTML includes JavaScript that runs in the browser.
  3. Calls: JavaScript makes asynchronous API calls to the server to fetch data (using AJAX).
  4. Rendering: The broswer processes the received data and dynamically updates the HTML to display the content.
